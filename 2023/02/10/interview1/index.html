组成原理

笔试的时候考了一点组成原理的基础知识，面试没问。

EBP（Extended Base Pointer，扩展基址指针寄存器），指向函数被调用时的栈顶。

ESP（Extended Stack Pointer，扩展栈指针寄存器），指向栈顶。

栈指针是从高位到低位移动的，栈内数据越多，ESP指向的地址就越小。

Extended表示寄存器是32位的。在32位机上，指针一次移动4字节。

冯·诺依曼型计算机的CPU工作阶段：取指、译码、执行、访存、回写。

  • 取指：将指令从主存中取到指令寄存器中
  • 译码：指令译码器对取回的指令进行拆分和解释
  • 执行：完成指令的操作
  • 访存：根据指令需要，从主存中取操作数进行运算
  • 回写：将运行结果写回寄存器或主存

RISC：Reduced Instruction Set Computer，精简指令集计算机，指令长度较短。

C++

笔试和面试都考了，可以说是重点

内存分配方式

从静态存储区分配：在编译时期就已经分配好，且这块内存在程序整个运行期间都存在，如全局变量、static 变量等。

在栈上创建：在函数执行的时候内部变量存在栈上，执行完毕后自动释放。栈内存分配运算内置于处理器的指令集中，效率很高，但是分配的内存容量有限。

在堆上分配，即动态内存分配，内存生存期由程序决定，如果不释放就会内存泄漏。（new、malloc 分配的内存都在这里）

一个C/C++程序占用的内存分为以下几个部分：

  1. 栈区（stack）：由编译器自动分配释放，存放为运行函数而分配的局部变量、函数参数、返回数据、返回地址等。其操作方式类似于数据结构中的栈。
  2. 堆区（heap）：一般有程序员分配释放，若程序员不释放，程序结束时可能由操作系统回收。分配方式类似于链表。
  3. 全局区（静态区）：存放全局变量、静态数据、常量、程序结束时系统回收。
  4. 文字常量区：常量字符串就是放在这个地方。程序结束时由系统释放。
  5. 代码区：存放函数体（类成员变量和全局函数）的二进制代码。

网络

区别 TCP UDP
连接 有连接 无连接
顺序 保证有序 不保证有序
可靠性 可靠传输 可能丢包
效率 慢 快
首部 大（20B） 小（8B）
分割 字节流，可分割 报文，不可分割

TCP 只能单播，而 UDP 可以多播、广播。

TCP 拥塞控制：慢启动后指数增长，达到阈值后加法增大（拥塞避免，防止拥塞窗口快速增大），快速重传（检测丢包，3个ACK，无须等计时器超时），丢包后快恢复（窗口减半）。
